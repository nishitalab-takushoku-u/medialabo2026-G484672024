「Webコンテンツ」作成の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的です。到達目標は次の4つです。(1) 手順書を読み、実験を行い、レポートやプレゼンテーションにより報告するという実験の一連の流れを実践できる、（2）コミュニケーション能力の重要性を理解し、グループワークを円滑に実施できる、（3）モーション解析・編集ができる、（4）HTML等を用いたWebページ作成ができる。なお、本授業は、リーダーシップ能力、課題に対する解決能力育成に寄与する科目です。

プロジェクト課題について

本プロジェクト課題ではWebAPIを活用し、選択した都市の天気情報を動的に表示するアプリケーションを作成した。HTMLのセレクトボックスで都市を選び、検索ボタンを押すと、JavaScriptによるAjax通信が実行される。取得したJSONデータから都市名、気温、湿度などの情報を抽出し、DOM操作によってページを再読み込みすることなく画面にリスト表示する仕組みを構築した。非同期通信とDOM操作を組み合わせた実践的な開発手法を学ぶことができた。

プロジェクト課題で作成したページ: project.html

数当てゲーム

この数当てゲームは、コンピュータがJavaScriptの乱数生成を利用してランダム作られた1~10までの隠された数字をプレイヤーが推測して当てるミニゲーム。プレイヤーはテキストボックスに予想した数値を入力し、回答ボタンを押すことで判定を行う。回答チャンスは最大3回まであり、間違えた場合でも、入力した数字に対して答えがより大きいか小さいかのヒントが画面に表示されるため、プレイヤーはそれを頼りに論理的に正解を絞り込むことができる。3回以内に当てることができればクリア、できなければゲームオーバーとして答えが表示される。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

インターネット上で情報がやり取りされるWebの仕組みと、Webページを作成するための基本的なマークアップ言語であるHTMLについて学習した。タグを用いて見出しや段落、リスト、画像、ハイパーリンクなどの各要素を適切に記述する方法や、文書を論理的に構造化することの重要性を学んだ。これにより、Webブラウザ上で情報を正しく表示させるための基礎的な技術を身につけることができた。

演習の成果: 演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

HTMLで作成された文書に対して、CSSを用いて視覚的な装飾を施す方法を学んだ。文字色やフォントの基本設定、余白の調整、グリッドなどのレイアウト配置を実践し、デザイン性の高いWebページを構築するための基礎技術を習得した。

演習の成果: 演習2-3:ウェブブラウザの紹介

第3回 JavaScript の基本

変数や配列、オブジェクトなどJavaScriptの基礎的な構文を学習した。データ構造の理解を通じ、Webページへの動的な処理を実装するためのプログラミングの基本的な概念と、データの扱い方を実践し、論理的な記述方法を習得した。

演習の成果: 演習3-6:JavaScriptのオブジェクト

第4回 DOM と JavaScript

JavaScriptを用いてWebページを操作するDOMとイベント処理について学習した。要素の検索やinput要素からの値の取得、テキストの変更方法を学び、ボタンのクリックなどユーザーの操作に応じた動的な処理を実装する技術を習得した。

演習の成果: 演習4-10:足し算ページ

第5回 イベント処理

JavaScriptを用いたDOMの高度な機能について学習した。要素を検索してテキストを変更するだけでなく、新しいHTML要素を新規作成したり、既存の要素を編集・削除したりする方法を学習した。ページの構造を動的に変化させる、より実践的なDOMプログラミングの技術を習得した。

演習の成果:演習5-7:DOMプログラミング(2)

工夫点，苦心点，強調点，感想

ユーザにとって直感的に操作でき、かつ見やすくなるようなWebページになるよう工夫した。cssを用いて枠や色を付けデザインに統一感を持たせた。文字色や背景色をネイビーと白を中心にまとめ、検索結果の目立つ部分にはアクセントとして明るい黄色を使うことで、情報がきれいに見えるように工夫した。またJavaScriptの処理においても入力されたデータをDOM操作で追加して新しく要素を作って画面に追加して表示し、検索ボタンを押すたびに過去の検索結果が無限に蓄積するのを防ぐためにremove文を用いて古い検索結果を削除する処理を追加した。これにより、何度検索しても常に最新の結果だけを画面に表示させることができるようになった。 苦労した点はコードの書き込み。htmlやcssはこれまで使ったことがほとんどなかったこともあって、説明を聞いてもうまくいかず、レイアウトが崩れたりなどとても苦戦した。また、JavaScriptでのAjax通信やDOM操作も扱ったことがなく、新しく出てくる関数やメソッドを一つずつ覚えていくのが大変だった。しかし、コンソールやデバッガーなどのウェブ開発ツールの使い方を覚えていくにつれて、赤いエラー文(is not definedなど)が出ても、それが何行目のどんなミスなのか、どこをどう直せばよいのかすぐに理解することができるようになった。このツールのおかげで後半は間違いをすぐに直せるようになり、作業の効率を大きく上げることができた。 プログラムの動的な処理と、見た目のデザインの両立ができたこと。javaScriptによるcreateElementやappendChildを使った要素の生成を使いつつ、そこにh2やul,liなどのhtmlのタグを適切に割り当てていった。そして、ただ文字を出すだけでなく、cssで設定したクラスや枠のデザインがしっかり適応されるように作った。このようにhtmlで骨組みを作り、JavaScriptでデータを取得して画面を更新し、cssできれいにデザインするという、Web開発の一連の流れの知識を学ぶことができた点が今回の課題の強調点だと思う。 感想：最初はわからなくて難しかったAPI通信やDOM操作も、一つずつデバッガーでエラーを解消し、自分で書いたコードが実際のブラウザの画面上でうまく動作しているのを見て、プログラミングの楽しさ、うまくいったときの嬉しさを体感することができた。project.htmlとcss、weather.jsが完成した際の達成感はとても大きかった。エラーが出ても落ち着いて対処して修正できる力も身に付いた。今回学んだWebページ作成の基礎を忘れずに、これからもいろいろなhtmlのプログラミングを組んだみたいと感じた。
